# Веб-консоль управления (frontend)

Консоль управления для **guardrails-llm-filter**: SPA на React 18 + TypeScript + Vite.
Она **вшита в бинарь сервиса** и отдаётся с origin'а management API (`:9080`), поэтому
консоль и её API живут на одном origin и поставляются одним Docker-образом — без
отдельного веб-сервера и без CORS.

Собрана на компонентной библиотеке Cloud.ru
[`@snack-uikit`](https://github.com/cloud-ru-tech/snack-uikit).

## Что делает

Разделы, все работают через `/v1/*` API на `:9080`:

- **Правила** — просмотр встроенных и кастомных правил детекции, фильтр по
  источнику / типу данных / тексту, включение/выключение любого правила, создание /
  редактирование / удаление кастомных правил. Валидация правил — на сервере (API
  компилирует RE2-regex и валидаторы); детали отказа показываются прямо в форме.
- **Настройки** — глобальная политика: главный переключатель, режим enforce/detect и
  какие категории типов данных участвуют в сканировании.
- **Аудит** — пофазовый аудит-трейл маскирования (какие правила и типы данных сработали и
  какие плейсхолдеры заменили найденные значения — оригиналы не хранятся, если оператор
  не включил это явно). С фильтрами, курсорной пагинацией и детальным видом записи.

## Архитектура

Браузер всегда обращается только к относительным путям `/v1/...` того же origin
(`src/api/client.ts` использует `baseUrl: ''`). В рантайме Go-сервис отдаёт собранную SPA
на `/`, а её API — на `/v1/*` на том же management-порту, поэтому CORS и клиентский URL
бэкенда не нужны.

```
Браузер ──/       ──► guardrails-llm-filter :9080  (вшитая SPA, SPA-fallback)
        ──/v1/*   ──► guardrails-llm-filter :9080  (REST API grpc-gateway)
```

Management-порт **неаутентифицирован по замыслу** — держите его внутри кластера (никогда
публичный ingress). Отдача консоли переключается `GUARDRAILS_UI_ENABLED` (по умолчанию
`true`).

## Разработка

Два процесса: Go-сервис (API) и dev-сервер Vite (hot reload). Dev-сервер Vite проксирует
`/v1` в Go-сервис, поэтому вы получаете ту же same-origin-модель, что и в проде.

```sh
# 1) запустить сервис с включённым management API
cd .. && make build
GUARDRAILS_API_ADDR=:9080 GUARDRAILS_AUDIT_ENABLED=true ./bin/guardrails-llm-filter

# 2) запустить dev-сервер консоли
cd frontend
npm install
cp .env.example .env.local      # задать GUARDRAILS_API_URL=http://localhost:9080
npm run dev                     # http://localhost:5173
```

## Скрипты

| Команда          | Назначение                                                  |
| ---------------- | ----------------------------------------------------------- |
| `npm run dev`    | dev-сервер Vite с прокси `/v1`                              |
| `npm run gen`    | перегенерировать `src/api/schema.d.ts` из `spec/openapi.yaml` |
| `npm run build`  | проверка типов + прод-сборка в `dist/` (сначала запускает `gen`) |
| `npm run lint`   | ESLint                                                      |
| `npm run format` | Prettier                                                    |

## Как поставляется

- **Docker (один образ)**: в корневом `Dockerfile` есть стадия `node:22-alpine`, которая
  запускает `npm run build`; результат копируется в `frontend/dist` и вшивается в
  Go-бинарь через `//go:embed` (см. `frontend/embed.go`). Собрать всё целиком —
  `make docker-build` из корня репо.
- **Локальный бинарь**: выполните `make frontend` (из корня репо), чтобы наполнить
  `frontend/dist`, затем `make build`. Без сборки UI вшивается только закоммиченный
  плейсхолдер `dist/.gitkeep`, и сервис отдаёт только API.

## Контракт API

Типы генерируются из `spec/openapi.yaml`. При изменении management API обновите спеку и
запустите `npm run gen`.
